# 常见问题

## 什么是 MesaLogo？

MesaLogo 是一个融合大语言模型与传统智能体建模的创新平台，专注于模拟基于对话的智能体互动。它通过双引擎规则系统、监督者机制和多智能体协作框架，为复杂多方协作决策提供智能化解决方案。

## MesaLogo 与传统 ABM 软件有什么区别？

传统 ABM 软件（如 NetLogo、Mesa、AnyLogic）主要关注空间移动和状态变化，需要编程技能来定义规则。MesaLogo 则以对话和沟通为中心，支持自然语言规则定义，内置自动化监督机制，适合技术和非技术用户使用。

主要区别：
- **规则定义**：MesaLogo 支持自然语言规则 + 程序逻辑规则双引擎
- **用户门槛**：适合技术和非技术背景用户
- **监督机制**：内置自动化监督者角色
- **交互重点**：以对话和沟通为中心

## MesaLogo 与现代 LLM 平台有什么区别？

现代 LLM 平台（如 Dify、Langflow、RAGFlow）主要面向单一智能体或简单的多轮对话，关注知识检索和工作流自动化。MesaLogo 则专注于复杂多方交互场景。

主要区别：
- **多智能体协作**：内置多角色协作框架
- **规则系统**：双引擎混合规则系统
- **环境变量**：完整的环境变量架构
- **应用重点**：复杂多方交互场景

## 双引擎规则系统是如何工作的？

双引擎规则系统结合了自然语言规则引擎和逻辑规则引擎：

- **自然语言引擎**：处理复杂的语义理解和对话逻辑
- **逻辑规则引擎**：负责精确的计算和条件判断

两个引擎协同工作，既保证了系统的灵活性，又确保了执行的准确性。

- 红蓝军演
- 模拟法庭
- 软件开发协作

任何需要多方协作决策和对话交互的复杂场景都可以通过 MesaLogo 进行建模和模拟。

## MesaLogo 需要什么技术背景？

MesaLogo 设计为适合不同技术背景的用户：

- **非技术用户**：可以通过自然语言定义规则和场景
- **技术用户**：可以深度定制逻辑规则和系统集成

基本的计算机操作能力即可开始使用，高级功能可能需要一定的编程知识。

## MesaLogo 的监督者机制如何保证模拟质量？

监督者机制通过以下方式保证模拟质量：

- **实时监控**：监控智能体行为、规则执行情况和对话质量
- **异常检测**：自动检测异常行为
- **动态反馈**：提供实时反馈
- **流程调整**：调整对话流程
- **必要干预**：在必要时进行干预

这确保了模拟过程的合理性和结果的可靠性。

## MesaLogo 支持与外部系统集成吗？

是的，MesaLogo 通过 MCP（Model-Control-Protocol）插件系统支持与外部系统的集成。

支持的集成类型：
- 真实的业务系统
- 数据库
- API 服务
- 智能设备
- 文件系统

这使得智能体可以将模拟扩展到实际的行动领域，实现更真实和有价值的应用场景。

## MesaLogo 支持哪些大语言模型？

MesaLogo 支持多种大语言模型和平台：

- OpenAI (GPT-3.5, GPT-4, etc.)
- Anthropic Claude
- Google Gemini

同时兼容外部智能体平台：

## MesaLogo 的性能如何？
